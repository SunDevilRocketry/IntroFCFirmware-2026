## USB Driver Module

The USB driver (`usb.c`/`usb.h`) abstracts the STM32 UART HAL layer, providing 
two communication modes of blocking or interrupt-driven. 

### Features
- **Dual-mode operation**: Both blocking and interrupt-driven communication can be utilized at the same time. Interrupt can be disabled during initialization.
- **Simple API**: Functions are documented both in code and here for simple integration. 
- **Error handling**: Status returns (OK, FAILED, TIMEOUT) are offered.
- **Zero-copy option**: Direct buffer access available. 

### Important Notes
Dependencies:
(1) CubeMX generated configuration (provides clock, GPIO, and UART setup)
[!IMPORTANT]
When using Interrupt Mode, ensure that UART Global Interrupts are enabled in the STM32CubeMX NVIC settings. If interrupts are disabled in hardware, the background reception functions will not trigger.

For future reference, the UART handle being used for testing/documentaiton is **huart3** which is generated by CubeMX ensure you modify usb_init() parameter to match your configuration.

This driver is designed for the STM32H7 series but can be adapted for other STM32 families

### Mode Selection
Typically, this driver can be used in 'main.c'

Initialize the driver with your preferred mode:


OPTION 1: FULL Feature Mode │  Interrupt + Blocking modes available
```c
usb_init(&huart3, true);   // Recommended for most applications
```

OPTION 2: BLOCKING-ONLY   Only blocking functions available
```c
usb_init(&huart3, false);  // For simple polling-only designs
```

The following are the segmented functions:

Blocking Mode Functions (Simple)
```c
USB_STAT usb_transmit(uint8_t* data, uint16_t size, uint32_t timeout_ms);
USB_STAT usb_receive(uint8_t* data, uint16_t size, uint32_t timeout_ms);
```
Function	                    Description
usb_transmit(...)	            Sends data; waits until completion or timeout.
usb_receive(...)	            Receives exact bytes; waits until completion or timeout.


Interrupt Mode Functions (Advanced):
```c
USB_STAT usb_IT_transmit(uint8_t* data, uint16_t size, uint32_t timeout_ms);
USB_STAT usb_IT_receive(uint8_t* data, uint16_t size, uint32_t timeout_ms, uint16_t* bytes_received);
uint8_t usb_data_available(void);
USB_STAT usb_get_received_buffer(uint8_t** data, uint16_t* size);
void usb_clear_buffer(void);
```
Function	                    Description
usb_IT_transmit(...)	        Non-blocking transmit with completion check.
usb_IT_receive(...)	            Receives line-based data (newline terminated).
usb_data_available()	        Returns 1 if new data has arrived (non-blocking).
usb_get_received_buffer()	    Provides zero-copy access to the internal buffer.
usb_clear_buffer()	            Manually resets/clears the receive buffer.



Where **USB_STAT** can be checked for any errors, defined as:
```c
typedef enum {
    USB_OK = 0,           // Operation successful
    USB_CRIT_FAILED,      // Critical failure (hardware error, invalid params)
    USB_TIMEOUT           // Operation timed out
} USB_STAT;
```

For robustness, checking **USB_STAT** for each return function is recommended, but not 'required'. The following generally OMITS such checks, with few exceptions for showcase purpose.

Sample code for blocking mode is provided as (simple/advanced mode):
```c
#include "usb.h"

// Initialize in blocking-only mode
usb_init(&huart3, false);

// Transmit data (waits for completion)
uint8_t message[] = "Hello World\r\n";
USB_STAT status = usb_transmit(message, strlen((char*)message), 100);
if (status == USB_OK) {
    // Transmission successful
}

// Receive exactly 10 bytes (waits until received or timeout)
uint8_t buffer[10];
if (usb_receive(buffer, 10, 1000) == USB_OK) {
    // Process exact 10-byte command
    process_command(buffer, 10);
}
```



Sample code for interrupt mode is provided as (advanced mode only):
```c
#include "usb.h"

int main(void) {
    HAL_Init();
    MX_USART3_UART_Init();
    
    // Initialize with interrupt mode enabled
    usb_init(&huart3, true);
    
    // Send welcome message
    usb_IT_transmit((uint8_t*)"Ready\r\n", 7, 100);
    
    while(1) {
        // Check if data received
        if (usb_data_available()) {
            uint8_t buffer[64];
            uint16_t len;
            
            // Get the received data
            if (usb_IT_receive(buffer, sizeof(buffer), 0, &len) == USB_OK) {
                // Echo back what was received
                usb_IT_transmit(buffer, len, 100);
            }
        }
        
        // Other tasks can run while waiting for data
        HAL_Delay(10);
    }
}
```
